# Definition of Done

## 기능 완료 기준

### 코드 품질
- [ ] 모든 코드가 TypeScript/Python 타입 체크를 통과
- [ ] Linter 오류 없음
- [ ] 코드 포맷팅 적용 (Black, Prettier)

### 테스트
- [ ] 단위 테스트 작성 및 통과 (커버리지 80% 이상)
- [ ] 통합 테스트 작성 및 통과
- [ ] E2E 테스트 작성 및 통과 (필요한 경우)

### 문서화
- [ ] API 문서 업데이트 (필요한 경우)
- [ ] 코드 주석 작성 (복잡한 로직)
- [ ] README 업데이트 (필요한 경우)

### 기능 검증
- [ ] 요구사항 명세서의 기능 요구사항 충족
- [ ] UI/UX 설계 명세서 준수
- [ ] 다크 모드 지원 (프론트엔드 컴포넌트)

### 성능
- [ ] 페이지 로딩 시간 3초 이내
- [ ] 차트 렌더링 시간 1초 이내
- [ ] 데이터 갱신 주기 30초 이내

### 보안
- [ ] 입력 데이터 검증
- [ ] SQL Injection 방지 (ORM 사용)
- [ ] 환경 변수 관리 (.env)

### 브라우저 호환성
- [ ] Chrome 최신 버전
- [ ] Firefox 최신 버전
- [ ] Safari 최신 버전
- [ ] Edge 최신 버전
