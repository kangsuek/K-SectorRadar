## 1. 문서 개요

### 1.1 목적
본 문서는 K-SectorRadar 프로젝트의 기능적 및 비기능적 요구사항을 명확히 정의하고, 개발 및 검증 기준을 제시하기 위해 작성되었다.

### 1.2 범위
본 문서는 K-SectorRadar 시스템의 기능적 요구사항과 비기능적 요구사항을 포함한다.

### 1.3 참고 문서
- 프로젝트 개발 욕구사항 및 기능 명세서 초안

### 1.4 프로젝트 개요

#### 1.4.1 프로젝트 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | K-SectorRadar |
| 부제 | 한국 고성장 섹터 분석 |
| 프로젝트 유형 | 웹 애플리케이션 |
| 목적 | 사용자가 등록한 종목에 대한 종합 분석 및 리포팅 서비스 제공 |

- 사용자 친화적인 대시보드 인터페이스 제공
- 종목 간 비교 분석 기능 제공

## 2. 기능적 요구사항 (Functional Requirements, FR)

### 2.1 데이터 수집 요구사항 (FR-DTS)

| ID | 항목 | 설명 |
|:---|:---|:---|
| **FR-DTS-01** | 데이터 소스 | **가격 데이터**: Naver Finance, **매매 동향**: Naver Finance, **뉴스 데이터**: Naver News를 데이터 소스로 사용한다. |
| **FR-DTS-02** | 수집 대상 종목 (초기) | **ETF 4개**: 삼성 KODEX AI전력핵심설비 ETF (487240), 신한 SOL 조선TOP3플러스 ETF (466920), KB RISE 글로벌원자력 iSelect ETF (442320), KoAct 글로벌양자컴퓨팅액티브 ETF (0020H0) / **주식 2개**: 두산에너빌리티 (034020), 한화오션 (042660)를 초기 수집 대상으로 한다. |
| **FR-DTS-03** | 데이터 수집 주기 | **자동 갱신**: 30초 간격 (기본값)으로 데이터를 갱신한다. |
| **FR-DTS-04** | 수동 갱신 | 사용자의 요청 시 수동으로 모든 데이터를 갱신하는 기능을 제공해야 한다. |

### 2.2 페이지 및 라우팅 요구사항 (FR-PG)

| ID | 경로 | 페이지명 | 설명 |
|:---|:---|:---|:---|
| **FR-PG-01** | / | Dashboard | 메인 대시보드. 종목 카드를 통한 모니터링 기능 제공. |
| **FR-PG-02** | /etf/:ticker | Detail | 종목 상세 분석 페이지. |
| **FR-PG-03** | /compare | Comparison | 종목 간 비교 분석 페이지. |
| **FR-PG-04** | /settings | Settings | 시스템 설정 페이지. |
| **FR-PG-05** | 네비게이션 | 헤더 메뉴 구성 | 로고(홈 이동), Dashboard, Comparison, Settings 링크 및 외부 GitHub 링크를 제공한다. |

### 2.3 Dashboard 페이지 요구사항 (FR-DB)

| ID | 항목 | 상세 요구사항 |
|:---|:---|:---|
| **FR-DB-06** | 헤더 영역 | "SectorRadar Dashboard" 제목 및 "총 N개 종목" 카운트를 표시한다. |
| **FR-DB-07** | 필터 및 정렬 | 타입순(기본 선택), 이름순, 테마순, 등락률순(높은순/낮은순), 거래량순(높은순/낮은순) 정렬 기능을 제공한다. (UI 형태: 드롭다운 또는 탭) |
| **FR-DB-08** | 업데이트 정보 | "YYYY년 MM월 DD일 요일" 형식의 날짜, "HH:MM:SS" 형식의 마지막 수집일시 및 마지막 업데이트 일시를 표시한다. |
| **FR-DB-09** | 자동 갱신 토글 | 체크박스 형태의 "자동 갱신 (30초)" 토글 기능을 제공하며, 기본값은 체크됨(활성화)이다. |
| **FR-DB-17** | 새로고침 버튼 | 수동으로 모든 데이터를 갱신하는 새로고침 버튼을 제공한다. |
| **FR-DB-10** | 종목 카드 (일반) | 종목명, 타입 뱃지(STOCK/ETF), 테마 정보, 종목 코드를 표시한다. |
| **FR-DB-11** | 종목 카드 (가격) | 현재가(큰 글씨), 등락률(빨강/파랑 색상 구분), 시가, 고가, 저가, 거래량(단위 표시), 주간 등락률을 표시한다. |
| **FR-DB-12** | 종목 카드 (차트) | 6일간 가격 변동을 표시하는 작은 캔들스틱 차트를 포함한다. |
| **FR-DB-13** | 종목 카드 (매매 동향) | "매매 동향 (YYYY-MM-DD)" 형식의 날짜 표시 및 개인, 기관, 외국인 투자자별 거래량(금액 및 부호)을 색상 구분(파랑: 마이너스, 빨강: 플러스)하여 표시한다. |
| **FR-DB-14** | 종목 카드 (뉴스) | 뉴스 개수 ("N건") 및 최신 뉴스 헤드라인 1개를 표시한다. 종목 코드는 카드 하단에 표시한다. |
| **FR-DB-15** | 종목 카드 (ETF) | ETF의 경우 "수수료: X.XXXX%" 형식으로 수수료 정보를 표시한다. |
| **FR-DB-16** | 카드 인터랙션 | 카드 클릭 시 해당 종목의 상세 페이지로 이동하며, 호버 시 그림자 효과를 적용한다. |

## 3. 비기능적 요구사항 (Non-Functional Requirements, NFR)

| ID | 항목 | 상세 요구사항 |
|:---|:---|:---|
| **NFR-PER-01** | 성능 (갱신) | 데이터 갱신 주기: 최소 30초를 만족해야 한다. |
| **NFR-PER-02** | 성능 (로딩) | 페이지 로딩 시간: 3초 이내, 차트 렌더링 시간: 1초 이내를 목표로 한다. |
| **NFR-USE-01** | 사용성 | 직관적인 사용자 인터페이스 및 **반응형 웹 디자인**을 지원하여 모바일 및 데스크톱 환경에서 사용 가능해야 한다. |
| **NFR-SEC-01** | 보안 | 외부 API 호출 시 에러 처리 및 사용자 데이터 보호 방안을 마련해야 한다. |
| **NFR-CMP-01** | 호환성 | 최신 버전의 주요 웹 브라우저 (Chrome, Firefox, Safari, Edge)를 지원해야 한다. |
| **NFR-MNT-01** | 유지보수 | 코드 주석 및 문서화, 모듈화된 구조, 확장 가능한 아키텍처를 갖춰야 한다. |

## 4. 제약사항

### 4.1 기술적 제약사항
- 데이터 소스: Naver Finance, Naver News API 호출 제한 및 웹 스크래핑 제약을 준수해야 한다.
- 수집 주기: API 호출 제한을 고려하여 수집 주기를 조정해야 한다.

### 4.2 비즈니스 제약사항
- 초기 종목 수: 6개 (ETF 4개, 주식 2개)로 제한된다.
