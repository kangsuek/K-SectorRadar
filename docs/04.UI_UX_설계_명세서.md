## 1. 디자인 원칙

### 1.1 기본 원칙
- **명확성**: 데이터 중심의 직관적이고 명확한 정보 전달
- **일관성**: 전체 페이지에서 일관된 디자인 언어 사용
- **접근성**: 다양한 사용자 환경에서 접근 가능
- **반응성**: 빠른 피드백과 부드러운 인터랙션
- **Mobile First**: 모바일 환경을 우선 고려한 반응형 디자인

### 1.2 디자인 컨셉
- 모던하고 미니멀한 디자인
- 데이터 중심의 정보 전달
- 직관적인 네비게이션
- 효율적인 정보 계층 구조

### 2.5 다크모드 지원

#### 2.5.1 테마 토글
- **위치**: 헤더 (오른쪽, GitHub 링크 앞)
- **아이콘**: 태양/달 아이콘 (Material Icons 또는 Heroicons)
- **인터랙션**: 클릭 시 라이트모드와 다크모드 전환
- **지속성**: 사용자 선호도 localStorage에 저장
- **기본값**: 시스템 선호도 따름 (prefers-color-scheme 미디어 쿼리)

#### 2.5.2 전환 효과
- **지속 시간**: 테마 전환 시 300ms 부드러운 전환
- **범위**: 모든 색상 속성이 부드럽게 전환
- **성능**: 효율적인 테마 전환을 위해 CSS 커스텀 속성(CSS 변수) 사용

#### 2.5.3 색상 대비
- **WCAG AA 준수**: 라이트모드와 다크모드 모두에서 4.5:1 이상의 대비율 유지
- **텍스트 가독성**: 모든 텍스트가 두 테마에서 읽기 가능하도록 보장
- **주식 색상**: 다크모드에서 가시성 향상을 위해 상승/하락 색상 조정

## 6. 반응형 디자인 (Mobile First)

### 6.1 브레이크포인트

| 디바이스 | 화면 너비 | 종목 카드 배치 |
|----------|-----------|----------------|
| **Mobile** | 0px ~ 768px | **1열 배치** |
| **Tablet** | 768px ~ 1024px | **2열 배치** |

- 카드: 1열 배치

- 터치 친화적 버튼 크기 (최소 44px)

### 6.3 태블릿 대응
- 카드: 2열 배치
- 간격 조정
- 중간 크기 폰트 유지

### 6.4 데스크톱 대응
- 카드: 3열 배치
- 최대 너비: 1400px (중앙 정렬)
- 최적화된 간격 및 여백

## 7. 접근성 (Accessibility)

### 7.1 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 접근 가능
- Enter/Space로 버튼 클릭
- Esc 키로 모달/드롭다운 닫기

### 7.2 스크린 리더 지원
- 적절한 ARIA 레이블 사용
- 시맨틱 HTML 사용 (header, nav, main, section, article 등)
- alt 텍스트 제공 (이미지, 아이콘)

### 7.3 색상 대비
- WCAG AA 기준 준수 (4.5:1 이상)
- 텍스트와 배경 간 충분한 대비 확보
- 색상만으로 정보를 전달하지 않음 (텍스트, 아이콘 병행)

## 10. 다크모드 구현 가이드라인

### 10.1 기술적 구현
- **CSS 변수**: 모든 색상 값을 CSS 커스텀 속성으로 사용
- **클래스 기반 토글**: 루트 요소(html/body)에 `dark` 클래스 추가/제거
- **상태 관리**: 테마 선호도를 localStorage에 저장
- **시스템 선호도**: 초기 로드 시 `prefers-color-scheme` 미디어 쿼리 감지 및 준수

### 10.2 컴포넌트 적응
- 모든 컴포넌트는 라이트모드와 다크모드를 지원해야 함
- 하드코딩된 색상 대신 시맨틱 색상 토큰 사용 (예: `--color-background`, `--color-text-primary`)
- 모든 인터랙티브 상태(호버, 활성, 포커스)를 두 테마에서 테스트

### 10.3 차트 및 데이터 시각화
- 차트는 다크모드에 맞게 색상이 조정되어야 함
- 데이터 포인트가 명확하게 보이고 구분 가능하도록 보장
- 차트 유형에 맞는 적절한 색상 스킴 사용

### 10.4 테스트 요구사항
- 두 테마에 대한 시각적 회귀 테스트
- 접근성 테스트 (대비율) 양쪽 모드에서 수행
- 크로스 브라우저 호환성 검증
- 테마 전환 애니메이션 성능 테스트
